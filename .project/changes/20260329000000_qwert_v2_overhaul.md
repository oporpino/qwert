# QWERT v2 — Overhaul Completo

> TLDR: Migrar qwert de shell scripts para Rust, introduzir sistema de recipes versionadas, config YAML declarativa, detecção automática de plataforma com overrides, e UI verbosa com ticks.

**Status:** in_progress
**Created:** 2026-03-29
**Owner:** @gporpino

---

## Context

O qwert hoje é uma coleção de shell scripts acoplados a macOS sem coesão arquitetural. A migração para Rust traz tipagem, testabilidade e UX rica. O sistema de recipes permite que ferramentas sejam declaradas no `setup.yml` e instaladas/atualizadas de forma idempotente. O `QWERT_CONFIG_DIR` (default `~/.config/qwert/`) é o dotfiles pessoal do dev — salvo em repo pessoal para replicar o ambiente em qualquer máquina.

## Objectives

- Migrar core para Rust
- `setup.yml` declarativo em `$QWERT_CONFIG_DIR`
- Sistema de recipes versionadas (TOML, tipo brew/qwert)
- Detecção automática de plataforma com overrides via trait
- UI verbosa: ✓ ok, → instalando, ✗ erro
- Comandos: add, remove, setup, status, search, list, upgrade, reinstall, update, doctor
- Autocomplete dinâmico via index.json do GitHub
- `qwert add script init/end --path` para gerenciar scripts zsh

## Changes

### Novos arquivos
- `src/` — código Rust completo
- `recipes/` — TOML definitions de cada tool
- `Cargo.toml`

### Modificados
- `scripts/install.sh` — verifica/instala Rust, compila, instala binário
- `CLAUDE.md` — documentação do QWERT_CONFIG_DIR concept
- `run` — gerado/mantido pelo binário Rust

### Removidos (após porte)
- `bin/qwert` (shell)
- `commands/**/*.sh`

## How to verify

```bash
# Build
cargo build --release

# Setup numa máquina nova
qwert setup
# → ✓ para tools instaladas, → para novas

# Add tool
qwert add tmux
qwert status tmux

# Search
qwert search neovim

# Upgrade
qwert upgrade homebrew
```
